Running a Full Node


This tutorial will go over the basics of spinning up a ltzcore node. Before you begin, you'll need to have around 8GB of memory and about 20GB~ of disk space available to store the litecoinz blockchain plus additional database information. Both 64bit Mac OS X and GNU/Linux are currently supported. The process of downloading the blocks and indexing can take upwards of 8 hours on livenet , depending on Internet connection and other factors such as CPU and disk speed. It's also possible to use an existing litecoinz data directory, which will speed up the process as reindexing can take upwards of 2 hours for livenet.



Install Node.js v4 LTS

It's recommended to install the Node Version Manager, as this makes it simple to switch between different Node.js versions. We will specifically need to install and run v0.12 or v4 LTS. Please follow the directions at [https://github.com/creationix/nvm#install-script](https://github.com/creationix/nvm#install-script) and then run:


nvm install v4


Install ZeroMQ and Tools


For GNU/Linux distribution such as Debian or Ubuntu:


apt-get install libzmq3-dev build-essential



For Mac OS X:


brew install zeromq


Install ltzcore

ltzcore comes with a command line utility for creating and managing your full node. To get started, run these commands, and you'll then have the `ltzcore` command in your path:



npm install -g ltzcore


Note: Do not run this command with `sudo` or with *root privileges* as this will lead to [permission issues](https://docs.npmjs.com/misc/scripts#user).



At this point, if you just want to sync the blockchain and run the Insight Block Explorer, then you are done!


Just run:


ltzcored


This will start your node from a config file in `~/.ltzcore` with the insight-api and insight-ui services enabled.



Create a New (Non-Global) Node for Running Custom Services like ltzcore Wallet Service

If you would like to create additional nodes that are not in your global npm installation directory, or run the ltzcore Wallet Service, please do the following:



ltzcore create mynode


or for testnet:


ltzcore create mynode --testnet


This will create the directory "mynode" in your current working and install all of the necessary dependencies and configuration files for your node.



To create a node if you already have a litecoinz data directory that you want to use:


ltzcore create -d mynode


The database directory will likely be at `~/.ltzcore`. Note that you'll need to have `txindex` enabled in your litecoinz configuration file. To enable `txindex`, add `txindex=1` and `reindex=1` to `~/.litecoinz/litecoinz.conf`. After you start litecoinz once with `reindex=1` you should remove it, otherwise it will reindex every time the node starts. Be sure to leave `txindex=1`, though.



For installation instructions regarding the ltzcore Wallet Service, please see [wallet service]



Configure the Network

Your node can run on "livenet" or "testnet". If you wish to configure the network, you can do so by opening the `ltzcore-node.json` configuration file with your favorite text editor (vi is used here):


cd mynode

vi ltzcore-node.json


Then change the network value to "testnet" or "livenet". Here is an example configuration file:


{

 "network": "livenet",

 "port": 3001,

 "services": [

  "litecoinzd",

  "web"

 ],

 "servicesConfig": {

  "litecoinzd": {

   "datadir": "/home/user/.litecoinz",

   "exec": "/home/user/litecoinz/src/litecoinzd"

  }

 }

}


Then change the network value to "testnet" or "livenet". Here is an example configuration file:






Note: This configuration includes an exec path to a locally compiled `litecoinzd` and shares the datadir with standard `.litecoinz` location.



Start Syncing

As mentioned previously, this process can take a few hours to complete, so you can start the script and come back later to check on the status. Let's get started!


cd mynode

ltzcored


This will start up all of the services that have been enabled in your configuration file. The first service that will most likely be started is litecoinz itself, followed by others that depend on it, such as the Database and Address Service. The syncing process will connect to other litecoinz peers in the network and start downloading the blockchain, verifying proof-of-work, and creating indexes for querying the blockchain. Both litecoinz and the Database Service will log the status of the initial synchronization process.



Query for Information



Get details about the genesis block:


ltzcore call getBlock 000025abeb658feda75c864877800064318a55161ce12c5e8d43b8ee4c7e0063


Get the details of a transaction (with address information):



ltzcore call getDetailedTransaction b786c135ad0ad91e5ae44acf9a38f11ea6f49ff3794ce12e821afae77c8f6577


Install a Block Explorer



Note: If you created your node by running `ltzcored` instead of `ltzcore create mynode`, your node will already have these services installed and you can skip this command.



While running individual CLI commands is a useful way to interact with your node, being able to view it in a GUI is even be better. Let's get started by installing the blockchain explorer Insight!


ltzcore install insight-api insight-ui


This will run an `npm` command to download the packages `insight-api` and `insight-ui` and add them to your node's `package.json` as well as `ltzcore-node.json`. The next time you start up your node, the services will be enabled, and you'll be able to open your web browser to view the explorer.



Go to the URL (default):



`http://localhost:3001/insight/`
